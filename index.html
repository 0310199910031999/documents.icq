API Documentation

1. POST http://localhost:4040/crm/setManager

Permite mandar en el cuerpo de la solicitud (Request Body) un JSON con los datos de un manager para ser registrado desde el servidor. Ejemplo de JSON:

{
    "id": 0,
    "nombre": "nombre de ejemplo",
    "direccion": "direccion de ejemplo",
    "edad": 0,
    "nEmpleado": 0,
    "rfc": "rfc de ejemplo"
}

No contiene parámetros en el URL.

Su código de respuesta en caso de realizar la operación correctamente es 200: OK.

2. GET http://localhost:4040/crm/getManager/{id}

Permite recibir en el cuerpo de la respuesta (Request Body) un JSON con los datos de un manager en específico. Ejemplo de JSON:

{
    "id": 0,
    "nombre": "nombre de ejemplo",
    "direccion": "direccion de ejemplo",
    "edad": 0,
    "nEmpleado": 0,
    "rfc": "rfc de ejemplo"
}

El parámetro {id} en el URL corresponde al ID del manager que se desea recibir en el cuerpo de respuesta. Ejemplo:

http://localhost:4040/crm/getManager/195

Respuesta:

{
    "id": 195,
    "nombre": "nombre del manager 195",
    "direccion": "direccion del manager 195",
    "edad": 0,
    "nEmpleado": 0,
    "rfc": "rfc del manager 195"
}

Su código de respuesta en caso de realizar la operación correctamente es 200: OK.

3. GET http://localhost:4040/crm/getManagers

Permite recibir en el cuerpo de la respuesta (Request Body) un arreglo de objetos JSON (JSON array) de todos los managers registrados. Ejemplo:

[
    {
        "id": 1,
        "nombre": "Juan Perez Hernandez",
        "direccion": "Linares 234 Azucenas Queretaro Queretaro",
        "edad": 24,
        "nEmpleado": 100,
        "rfc": "LAJA989596VA9"
    },
    .
    .
    .
    {
        "id": 2,
        "nombre": "Roberto Lino Ferrero",
        "direccion": "Begonias 25 Fuentes Queretaro Queretaro",
        "edad": 28,
        "nEmpleado": 250,
        "rfc": "LIRO958747LO9"
    }
]

No contiene parámetros en el URL.

Su código de respuesta en caso de realizar la operación correctamente es 200: OK.

4. PUT http://localhost:4040/crm/updateManager

Permite mandar en el cuerpo de la solicitud (Request Body) un JSON con los datos de un manager existente para ser actualizados desde el servidor. Ejemplo de JSON:

{
    "id": 1,
    "nombre": "nombre actualizado",
    "direccion": "direccion actualizada",
    "edad": 0,
    "nEmpleado": 0,
    "rfc": "rfc actualizado"
}

No contiene parámetros en el URL.

Su código de respuesta en caso de realizar la operación correctamente es 200: OK.

5. DELETE http://localhost:4040/crm/deleteManager/{id}

Permite mandar mediante un parámetro en el URL el ID de un manager registrado para ser eliminado. Ejemplo:

http://localhost:4040/crm/deleteManager/195

No contiene cuerpo ni en la solicitud ni en la respuesta.

Su código de respuesta en caso de realizar la operación correctamente es 200: OK.
